## 1. 프로젝트 개요

### 1.1 프로젝트 정의
- **프로젝트명**: 톡벨 (Tokbell) - 메시징 서비스 플랫폼
- **프로젝트 목적**: 개인 및 기업 회원이 문자(SMS/LMS/MMS)와 카카오톡(알림톡/브랜드 메시지) 메시지를 효율적으로 발송하고 관리할 수 있는 통합 메시징 서비스 플랫폼
- **핵심 가치**: 
  - 다양한 메시지 타입 지원 (SMS/LMS/MMS, 알림톡/브랜드 메시지)
  - 법적 규정 준수 (정보통신망법, 공직선거법 등)
  - 사용자 친화적인 인터페이스로 대량 발송 간편화
  - 실시간 발송 결과 추적 및 통계 제공
  - 안전한 결제 및 세금계산서 발행 지원

### 1.2 목표 사용자
- **주 사용자 그룹**: 
  - 개인 회원: 개인 사업자, 프리랜서, 소규모 사업자
  - 기업 회원: 중소기업, 대기업, 공공기관, 정당/후보자
- **사용자 페르소나**:
  - 마케팅 담당자: 대량 광고 메시지 발송, 고객 관리
  - 행정 담당자: 공지사항 발송, 이벤트 안내
  - 선거 캠페인 매니저: 공직선거 문자 발송
  - 고객 서비스 담당자: 알림톡을 통한 고객 안내

### 3.2 문자 발송

#### 3.2.1 일반문자 발송 (message-send-general.html)
- **기능**: SMS/LMS/MMS 메시지 작성 및 발송
- **화면 구성**: 2단 레이아웃 (입력 영역 + 미리보기)
- **주요 기능**:
  | 기능 | 설명 |
  |------|------|
  | 발신번호 선택 | 승인된 발신번호 드롭다운 |
  | 수신번호 입력 | 직접입력/주소록/엑셀 업로드 |
  | 메시지 작성 | 실시간 바이트 카운터, 변수 삽입 |
  | 이미지 첨부 | MMS 이미지 첨부 (최대 3개) |
  | 발송 시간 설정 | 즉시/예약 발송 |
  | 템플릿 | 템플릿 선택/저장 |
  | 테스트 발송 | 발송 전 테스트 |

#### 3.2.2 광고문자 발송 (message-send-ad.html)
- **기능**: 광고성 메시지 발송 (정보통신망법 준수)
- **추가 요소**:
  | 요소 | 설명 |
  |------|------|
  | (광고) 문구 | 자동 삽입, 옆 문구 입력 |
  | 080 수신거부 번호 | 필수 선택 (셀렉박스) |
  | 발송 시간 제한 | 평일 08:00~21:00 |
  | 법적 확인 | 체크박스 필수 선택 |

#### 3.2.3 선거문자 발송 (message-send-election.html)
- **기능**: 공직선거법 준수 선거 문자 발송
- **추가 요소**:
  | 요소 | 설명 |
  |------|------|
  | [선거] 문구 | 자동 삽입, 후보자/정당명 입력 |
  | 선거운동정보 | 선택사항, 최대 20자/40바이트 |
  | 080 수신거부 번호 | 필수 선택 |
  | 발송 시간 제한 | 07:00~23:00 |
  | 금지어 검증 | 허위사실, 비방 내용 검출 |
  | 수신번호 제한 | 숫자만 입력 가능 |

- **화면 구성**: 발신프로필 선택 → 템플릿 선택 → 변수 입력 → 발송

  | 기능 | 설명 |

| 브랜드 메시지 | 평일 08:00~21:00, 주말/공휴일 불가, 광고성 표시 필수, 080 수신거부 자동 포함 |

## 8. 용어 정의

| 용어 | 설명 |
|------|------|
| SMS | Short Message Service, 단문 메시지 (90바이트 이하) |
| LMS | Long Message Service, 장문 메시지 (2,000바이트 이하) |
| MMS | Multimedia Message Service, 멀티미디어 메시지 |
| 알림톡 | 카카오톡을 통한 정보성 메시지 (템플릿 승인 필수) |
| 브랜드 메시지 | 카카오톡을 통한 광고성 메시지 |
| 발신프로필 | 카카오톡 채널 (@아이디 형태) |
| 080 수신거부 | 광고성 메시지 수신거부 번호 |
| 포인트 | 메시지 발송에 사용하는 충전 금액 |
| 마일리지 | 보너스 포인트 (환불 불가) |
